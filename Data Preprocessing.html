Effective **Anomaly Detection** relies heavily on clean, structured, and relevant data. IoT sensor streams often suffer from noise, missing data points, and inconsistent scaling. Our robust preprocessing pipeline is designed to transform raw sensor data into an optimal format for Machine Learning models.

1. Data Cleaning and Handling Missing Values

  • **Noise Reduction:** We apply filtering techniques (like a moving average) to smooth out high-frequency sensor noise, ensuring the model focuses on systemic changes rather than random spikes.
  • **Missing Data Imputation:** Missing timestamp values, which occur frequently in network transmissions, are handled using time-series specific imputation methods (e.g., forward-fill or interpolation).
  • **Outlier Filtering:** While the goal is to *find* anomalies, non-systemic, extreme measurement errors (e.g., a momentary sensor failure) are identified and treated separately to prevent them from skewing the training phase.

2. Normalization and Scaling

  • **Standardization:** Different sensors (temperature, pressure, vibration) have different units and scales. We use **Z-Score Normalization** to standardize features, giving them zero mean ($\mu=0$) and unit variance ($\sigma=1$). This ensures all features contribute equally to the distance metrics used by the anomaly models.
  • **Time-Series Structuring:** Data is often aggregated into fixed-size windows (e.g., 5-minute intervals) and structured using **lagged features** to capture temporal dependencies, which is critical for time-series anomaly detection.

3. Feature Engineering

  • **Statistical Features:** New features are calculated based on the data window, including the minimum, maximum, mean, and standard deviation of each sensor reading within that window.
  • **Temporal Features:** We extract features like **day of the week** and **hour of the day** to capture natural, periodic variations in the IoT environment.
